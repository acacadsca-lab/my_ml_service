K L N College of Engineering

  • Core Objective: The project aims to design a voice-controlled home automation system that allows users to control household appliances (like lights and fans) using voice commands through Google Assistant on a smartphone or smart speaker.
  • Hardware Core (NodeMCU): The central controller used is the NodeMCU (ESP8266). This microcontroller is chosen because it has built-in Wi-Fi capabilities, allowing it to connect to the internet and receive remote commands.
  • Cloud Integration (Adafruit IO): The project utilizes Adafruit IO as a cloud-based MQTT broker. It acts as the bridge that stores the state of the devices (On/Off) and communicates data between the Google Assistant and the physical hardware.
  • Connectivity Bridge (IFTTT): To link Google Assistant to the Adafruit cloud, the researchers used IFTTT (If This Then That). IFTTT creates "applets" that trigger a specific action in Adafruit IO whenever a specific voice command is given to Google Assistant.
  • Control Mechanism: The system uses a Relay Module connected to the NodeMCU. The relay acts as an electromagnetic switch that allows the low-voltage microcontroller to safely control high-voltage (AC) home appliances.
  • Accessibility Focus: A primary motivation for the project is to assist elderly people and individuals with physical disabilities, enabling them to manage their environment without needing to move to manual switches.
  • Cost-Effectiveness: The paper highlights that by using open-source hardware (NodeMCU) and free software platforms (IFTTT, Adafruit IO), the system is significantly more affordable than commercial "smart home" products.
  • Remote Access: Because the system is cloud-based, it is not limited to local Wi-Fi range. A user can control their home appliances from anywhere in the world as long as they have an internet connection.
  • Programming Environment: The NodeMCU is programmed using the Arduino IDE. The code includes libraries for Wi-Fi connectivity and MQTT protocols to ensure stable communication with the Adafruit servers.
  • Future Scope: The authors suggest that the project can be expanded by integrating sensors (temperature, humidity, PIR) to automate the home further, such as turning on an AC automatically when a specific temperature is reached.
